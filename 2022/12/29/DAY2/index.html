刷题第二天

1.给你一个按 非递减顺序 排序的整数数组 nums，返回 每个数字的平方 组成的新数组，要求也按 非递减顺序 排序。

链接：https://leetcode.cn/problems/squares-of-a-sorted-array/

解法一：

最直观的思路，对数组的每个数字平方后然后排序即可得到新的排序数组

解法二：

这里我们需要知道，一个有序的数组，经过平方的之后得到的最大值也只可能从数组的两端获得，不可能是中间，然后我们可以使用双指针的操作，来每次获取数组的最大值，然后填充到与原数组等长里面即可。

2.给定一个含有 n 个正整数的数组和一个正整数 target 。

找出该数组中满足其和 ≥ target 的长度最小的 连续子数组 [numsl, numsl+1, …, numsr-1, numsr] ，并返回其长度。如果不存在符合条件的子数组，返回 0 。
链接：https://leetcode.cn/problems/minimum-size-subarray-sum

题解：

暴力解法：针对每一个游标i去找一个最少子串，找到后计算其长度，然后用result更新最小的长度。但是过数据的时候只能过一部分，算法超时也超内存。

在暴力解法中，是一个for循环滑动窗口的起始位置，一个for循环为滑动窗口的终止位置，用两个for循环 完成了一个不断搜索区间的过程。

解法二：是在动态的滑动左右窗口去寻找一个最优解，当窗口子数组的和大于等于target时，此时要注意记录长度和上次最佳结果进行比较，然后更新长度，然后这时候需要尝试寻找最优解，就是需要移动左游标，缩小窗口。当当窗口子数组的和小于target时，需要移动右游标，增大窗口。与解法一的区别是：我们中间使用while,满足sum>=target的次数肯定不是n级别。所以相比暴力解法，滑动窗口的做法是非常优秀的。

3.给你一个正整数 n ，生成一个包含 1 到 n2 所有元素，且元素按顺时针顺序螺旋排列的 n x n 正方形矩阵 matrix

链接：https://leetcode.cn/problems/spiral-matrix-ii/

题解：

这是一个数值模拟的问题，就按照要求模拟就可以，没有什么太大的技巧，就是边界的问题要弄清楚。
