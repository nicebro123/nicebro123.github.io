MYSQL必知必会1.聚簇索引和非聚簇索引MySQL的InnoDB索引数据结构是B+树，主键索引叶子节点的值存储的就是MySQL的数据行，普通索引的叶子节点的值存储的是主键值，这是了解聚簇索引和非聚簇索引的前提 聚簇索引就是主键的一种术语 什么是聚簇索引？很简单记住一句话：找到了索引就找到了需要的数据，那么这个索引就是聚簇索引，所以主键就是聚簇索引，修改聚簇索引其实就是修改主键。 什么是非聚簇索引？索引的存储和数据的存储是分离的，也就是说找到了索引但没找到数据，需要根据索引上的值(主键)再次回表查询,非聚簇索引也叫做辅助索引。 总结： 所以判断一个索引是不是聚簇索引就是观察其查询的字段需不需要回表查询，如果需要就是非聚簇索引。 主键一定是聚簇索引，唯一性索引则不一定，要看具体情况 举例说明(很重要)下面我们创建了一个学生表，做三种查询，来说明什么情况下是聚簇索引，什么情况下不是。 12345678create table student ( id bigint, no varchar(20) , name varchar(20) , address varcha ...
